Importation VMware - Virtualbox

Problèmes de compatibilité possible

Problèmes de disque dur

Si lors de l’importation de votre VM vous avez l’erreur :

Error reading "D:\VM-ZE5\export_fedora\FedoraAuto_Piquet_Thomas.ovf": Host resource of type "Other Storage Device (20)" is supported with SATA AHCI or virtio-scsi controllers only, line 50 (subtype:vmware.nvme.controller).

C’est que l’export des VM avec des disques NVMe crée des problèmes avec l’importation dans VMWare.

Dans le fichier .ovf de l’export remplacer : <rasd:ResourceSubType>vmware.nvme.controller</rasd:ResourceSubType>

par :

<rasd:ResourceSubType>AHCI</rasd:ResourceSubType>

Pour éviter ce type problème, utilisez des disques SATA pour vos VM VMWare.

Warning

Lorsque l’on fait ces modifications, il faut également changer le fichier .mf avec la nouvelle valeur sha256 pour le fichier .ovf. Une autre solution consiste simplement à effacer ce fichier .mf, dans ce cas VirtualBox ne fait pas de vérification.

Problèmes d’affichage

Si les VMware tools sont installés sur la VM, il peut y avoir des problèmes de compatibilité au niveau des pilotes graphiques.

Effectivement, le SE invité a les pilotes vmware et s’attend donc à une émulation de carte graphique VMWare, alors que Virtual va simuler un autre type de carte graphique. Pensez à changer la carte graphique émulée dans les paramètres de votre machine virtuelle VirtualBox.

Sélectionnez VMSAVGA dans Virtualbox pour une meilleure compatibilité.
