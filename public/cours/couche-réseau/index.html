Couche réseau

Structure d’une adresse IPv4

  • Hôtes, serveurs et équipements réseau utilisent des adresses IP pour s’identifier mutuellement.
  • Chaque adresse est une chaîne de 32 bits divisée en quatre parties appelées octets séparés par des points.
  • Chaque octet contient 8 bits (ou 1 byte) séparés par un point.

Notions d’arithmétique binaire pour le réseau

  • Le format binaire est un système de numération utilisant les chiffres 0 et 1 qui sont appelés des bits.
  • Système numérique décimale composé de chiffres 0 à 9.

Parties réseau et hôte de l’adresse IP

  • L’adresse IP est composé d’une partie réseau et d’une partie hôte.
  • Cette séparation est réalisée par le masque de sous-réseau.

Masque de sous-réseau

  • Les masques de sous-réseau sont des suites de 1 consécutif.
  • Ce qui permet l’écriture avec la longueur de préfixe

  • C’est grâce au masque de sous-réseau qu’un hôte peut déterminer si la destination est sur le même réseau que lui.
  • Pour déterminer l’adresse du réseau, on effectue une opération logique AND (&)

Multidifusion

En multidiffusion (multicast), le protocole IP utilise les adresses de 224.0.0.0 à 239.255.255.255, les 28 bits les moins significatifs constituent l’adresse du groupe.

Lorsqu’un poste veut envoyer un paquet à un groupe multicast, il envoie ce paquet à l’adresse IP identifiant ce groupe (par exemple : 224.1.2.3). La réception est réalisée par un routeur abonné au groupe et le paquet est alors dupliqué et renvoyé grâce à une trame de niveau 2 multicast.

Les adresses IPv4 privées ne sont pas uniques et peuvent être utilisées par n’importe quel réseau interne. Pour cette raison, elles ne sont pas routables globalement

Pour router ces adresses vers internet, on utilise le processus de traduction d’adresse réseau (NAT). Il convertit les adresses privées en adresse publique.

Adresses speciales

  • Adresse de bouclage : 127.0.0.1/8 -> utilisé pour vérifier la configuration TCP/IP
  • Adresse link-local : 169.254.0.0/16 -> utilisé en cas d’échec de la connexion avec un serveur DHCP

Structure d’une adresse IPv6

  • Espace d’adressage sur 128 bits (32 bits pour IPv4)
    • 340 282 366 920 938 463 463 374 607 431 768 211 456 -> 340 milliards de milliards de milliards de milliards d’adresses possibles.
  • Le format privilégié pour noter une adresse IPv6 est x:x:x:x:x:x:x:x , où chaque x est constitué de quatre valeurs hexadécimales.
    • Exemple : 2001:0DB8:0000:1111:0000:0000:0000:0200

Les adresses IPv6 utilisent aussi le concept de préfixe pour déterminer la partie réseau et la partie hôte. Généralement la longueur de préfixe recommandée est de 64 bits (/64).

Le processus de traduction d’adresse (NAT) devient inutile face au grand nombre d’adresses disponnibles.

Caractéristique de la couche réseau

  1. Adressage des terminaux - Les terminaux doivent être configurés avec une adresse IP unique pour l’identification sur le réseau.
  2. Encapsulation - La couche réseau encapsule l’unité de données de protocole (PDU) de la couche transport dans un paquet. Le processus d’encapsulation ajoute des informations d’en tête IP, telles que l’adresse IP des hôtes source et de destination.
  3. Routage - La couche réseau fournit des services pour diriger les paquets vers un hôte de destination sur un autre réseau. Pour voyager vers d’autres réseaux, le paquet doit être traité par un routeur . Le rôle du routeur est de sélectionner le meilleur chemin et de diriger les paquets vers l’hôte de destination. Ce processus est appelé le routage.
  4. Décapsulation - Lorsque le paquet arrive à la couche réseau de l’hôte de destination, l’hôte vérifie l’en tête IP du paquet . Si l’adresse IP de destination dans l’en tête correspond à l’adresse IP de l’hôte qui effectue la vérification, l’en tête IP est supprimé du paquet. Une fois la décapsulation effectuée par la couche réseau, la PDU de couche 4 est transmise au service approprié au niveau de la couche transport.

Exemple de protocoles de la couche réseau :

  • IPv4 ou IPv6
  • ICMP
  • ARP

Méthode de routage des hôtes

Un hôte peut envoyer un paquet à :

  • Lui-même : 127.0.0.1 (IPv4), ::1 (IPv6)
  • Hôtes locaux : la destination se trouve sur le même réseau local. Le commutateur (switch) est utilisé pour se connecter à l’hôte local.
  • Hôtes distants : la destination ne se trouve pas sur le même réseau local. Dans ce cas, on utilise la passerelle par défaut du réseau local.

C’est le périphérique source qui détermine si la destination est locale ou distante. Avec IPv4 cette détermination se fait avec le masque de sous-réseau.

Si un périphérique n’a pas de passerelle par défaut, son traffic ne peu pas quitter le réseau local. Autrement la table de routage du périphérique est configurée avec une route par défaut vers la passerelle.

La passerelle par défaut doit avoir une adresse IP dans le réseau local, et une autre vers internet.

Méthode de routage des routeurs

Les routeurs disposent, eux aussi, d’une table de routage. Cette table à 3 types d’entrées :

  1. Réseaux directement connectés - Ces entrées (routes) sont automatiquement ajoutées par le routeur, lorsqu’une interface est configurée avec une adresse IP et qu’elle est activée.
  2. Routes distantes - Ce sont les routes que le routeur n’a pas de connexion directe et peuvent être appris :
    • Manuellement - en ajoutant une route statique.
    • Dynamiquement - en utilisant un protocole de routage pour que les routeurs partagent leurs informations entre eux.
  3. Route par défaut - Comme un hôte, la plupart des routeurs comprennent également une entrée de route par défaut, une passerelle de dernier recours.
