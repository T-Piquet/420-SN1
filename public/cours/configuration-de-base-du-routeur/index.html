Configuration de base du routeur

Composants d’un routeur

Un routeur est un ordinateur. Comme les ordinateurs, un routeur nécessite un processeur, un système d’exploitation et de la mémoire.

Le système d’exploitation de Cisco est IOS (Internetworking Operating System).

Organisation de la mémoire des routeurs Cisco

Interfaces d’un routeur

  • Interfaces LAN et WAN
  • Interfaces de gestion

On accède à la gestion des routeurs par 3 moyens :

  1. Console : Port de gestion physique, on doit être en présence du routeur pour se connecter.
  2. SSH : Méthode de connexion sécurisé à distance.
  3. Telnet : Méthode de connexion non sécurisée à distance.

Configuration des Paramètres Initiaux du Routeur

On a un certain nombre de configurations à faire sur le routeur avant de commencer à paramétrer les tables de routage.

Les différents modes du CLI

Pour être capable de configurer le routeur, il faut être en mode privilégié. Ce mode est similaire à l’accès root de Linux.

Router>enable
Router#configure terminal
Router(config)#end
Router#disable
Router>

La commande enable permet le passage au mode privilégié, cela change le pompt du CLI de > à #.

Un mot de passe peut vous être demandé s’il a été configuré. Ce mot de passe peut être différent de celui autorisant la connexion au routeur.

Configuration de base du routeur

  • Configurer le nom de l’appareil
Router(config)# hostname hostname
  • Désactivez la recherche DNS.
Router(config)# no ip domain lookup
  • Sécuriser le mode d’exécution privilégié (mot de passe chiffré)
Router(config)# enable secret mot_de_passe
  • Cryptez tous les mots de passe en clair.
Routeur(config)# service password-encryption
  • Sécuriser le mode d’exécution de la console.
Router(config)# line console 0
Router(config-line)# password password
Router(config-line)# login
  • Sécuriser l’accès à distance Telnet et SSH.
Router(config)# line vty 0 4
Router(config-line)# password password
Router(config-line)# login
Router(config-line)# transport input {ssh | telnet}
  • Fournissez une notification légale
Router(config)# banner motd # message #
Router(config)# end
  • enregistrez la configuration une fois sorti du mode config
Router# copy running-config startup-config

Exemple de configuration :

Routeur (config) # hostname R1
R1 (config) # hostname R1
R1 (config) # no ip domain lookup
R1 (config) # enable secret class
R1 (config) # line console 0
R1 (config-line) # password cisco
R1 (config-line) # login
R1 (config-line) # line vty 0 4
R1 (config-line) # password cisco
R1 (config-line) # login
R1 (config-line) # transport input ssh
R1 (config-line) # exit
R1 (config) # service password-encryption
R1 (config) # banner motd
Enter TEXT message. End with a new line and the #
***************************************************
WARNING:
Unauthorized access is prohibited
***************************************************
R1 (config) # exit
R1 # copy running-config startup-config

On finit par enregistrer la configuration dans la NVRAM

Configuration des interfaces

L’étape suivante consiste à configurer les interfaces réseaux.

Configuration de l’interface G0/0/0

R1 (config) # interface gigabitEthernet 0/0/0
R1 (config-if) # description lien avec LAN
R1 (config-if) # ip address 192.168.0.1 255.255.255.0
R1 (config-if) # ipv6 address 2001:db8:acad:10::1/64
R1 (config-if) # no shutdown
R1 (config-if) # exit
R1 (config) #

Configuration de la passerelle par défaut

Cette dernière étape consiste à donner la première entrée à notre table de routage.

  • La passerelle par défaut est utilisée lorsque l’hôte veut transmettre un paquet à un périphérique situé sur un autre réseau.
  • L’adresse de la passerelle par défaut est généralement celle de l’interface du routeur reliée au réseau local de l’hôte.
  • Pour atteindre PC3, PC1 adresse un paquet avec l’adresse IPv4 de PC3, mais transfère le paquet à sa passerelle par défaut, l’interface G0/0/0 de R1.
